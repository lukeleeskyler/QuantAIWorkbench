# Security Policy

## API Keys

Do not commit `api_keys.env` or any real API key.

Use `api_keys.env.example` as a template, then create a local `api_keys.env` file for your own machine.

If you accidentally expose a key, revoke it immediately from the provider dashboard and create a new one.

## Reporting Issues

If you find a security issue, please open a private advisory on GitHub if available, or contact the maintainer privately.

Please do not publish working exploits or leaked credentials in public issues.
