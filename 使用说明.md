# Quant AI Workbench 使用说明

这是一个本地运行的“AI + 量化指标”交易分析工作台。它不会自动下单，也不会连接券商账户。

## 1. 启动工具

### Windows

双击或运行：

```bat
run.bat
```

### macOS / Linux

在终端进入解压目录后运行：

```bash
chmod +x run.sh
./run.sh
```

启动成功后，在浏览器打开：

```text
http://127.0.0.1:8787/
```

如果 8787 已经被占用，工具会自动尝试 8788、8789 等端口。请以终端里显示的实际地址为准，例如：

```text
Quant AI Workbench running at http://127.0.0.1:8788
```

## 2. 输入标的代码

在左侧“标的代码”输入框里输入代码，然后点击“分析”。

如果不确定代码是否正确，可以点击“查找代码”。工具会显示来自 Yahoo Finance 的候选代码，点击候选项即可分析。

常用示例：

- 美股：`AAPL`、`NVDA`、`SPY`
- 港股：`0700.HK`
- A 股：`600519`、`002859`、`300408`（也可以输入 `600519.SS`、`000001.SZ`）
- 日股：`7203.T`
- 英股：`AZN.L`
- 加密资产：`BTC-USD`、`ETH-USD`

## 3. 页面会展示什么

分析完成后，页面会展示：

- 数据健康：行情源名称、交易所、最后 K 线日期、样本数量和数据状态
- 最新价、涨跌幅、成交量、综合评分
- 买入观察、持有观察、卖出/回避等研究信号
- 短线、波段、中线三个时间段的观察信号
- K 线、MA20、MA60、BOLL
- RSI、MACD
- 20日、60日、120日表现
- 年化波动、ATR、量比
- 相关新闻资讯
- AI 或本地规则生成的分析报告

买卖信号是基于趋势、动量、RSI、MACD、均线、量能和风险评分生成的“观察信号”，用于辅助研究，不是自动交易指令。

如果数据源疑似查错代码或数据过期，页面会给出提醒。例如 `SPCX-USD` 可能被识别为代币而不是股票，工具会提示改查 `SPCX`。

## 4. 关注股票与自选股雷达

输入代码后点击“加入关注”，左侧会保存到“关注股票”。

关注列表保存在当前浏览器本地，不会上传，也不会写入 API key 文件。对方解压后使用时，会生成自己的关注列表。

点击关注列表里的代码，可以快速重新分析该标的。

点击“扫描”或页面里的“扫描关注”，工具会批量刷新关注列表，并显示“自选股雷达”：

- 当前价格和涨跌幅
- 综合评分与风险评分
- 买入观察、持有观察、等待确认、卖出/回避等信号
- 信号置信度
- 数据源提醒
- 最近信号变化记录

自选股雷达可以按综合评分、涨跌幅、风险评分或信号强度排序。信号变化记录保存在当前浏览器本地，用于观察某个标的是否从“等待确认”变成“买入观察”等状态。

## 5. 生成 AI 分析报告

点击“生成并保存”即可生成报告。

如果没有配置 API key，工具会使用本地规则生成报告，也可以正常使用。

如果想接入大模型，直接打开解压目录里的这个文件：

```text
api_keys.env
```

把你使用的平台 key 填进去，保存文件，然后重新运行 `run.bat` 或 `./run.sh`。

OpenAI 示例：

```text
OPENAI_API_KEY=你的 OpenAI key
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_MODEL=gpt-4.1-mini
```

DeepSeek 示例：

```text
DEEPSEEK_API_KEY=你的 DeepSeek key
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
DEEPSEEK_MODEL=deepseek-v4-flash
```

OpenRouter 示例：

```text
OPENROUTER_API_KEY=你的 OpenRouter key
OPENROUTER_BASE_URL=https://openrouter.ai/api/v1
OPENROUTER_MODEL=openai/gpt-4.1-mini
```

只需要填写你实际使用的平台，不需要全部填写。

## 6. 历史报告保存在哪里

报告会保存在本地：

```text
data/reports.sqlite3
```

左侧“历史报告”会显示最近保存的报告。

## 7. 注意事项

- 第一次分析需要联网访问行情和新闻数据。
- 行情和新闻来自 Yahoo Finance，部分市场或代码可能没有数据。
- 多时间段买卖信号是基于日线数据推导的观察信号，不代表实时盘口或自动交易策略。
- `api_keys.env` 里可能包含你的 API key，不要把填好真实 key 的文件发给别人。
- `.env.example` 只是兼容旧方式的模板，普通用户可以忽略。
- 本工具仅供研究和信息分析，不构成投资建议或交易指令。
